# Phase1 解像度固定ポリシー反映プラン

## 目的

- 16:9 固定の解像度リスト（1920x1080 / 1600x900 / 1280x720）を設計に明記し、起動時はリスト内で最大の解像度に固定する。
- ウィンドウリサイズを禁止し、将来の設定メニュー経由でのみ解像度変更を許可する方針を実装に反映する。

## 変更範囲

- 設計ドキュメント: Phase1設計（解像度/ウィンドウポリシー追記）
- 実装: 起動時の解像度決定・Windowフラグ、Render/入力スケールは現行の仮想FHDを維持

## 手順

1) 設計更新

- Phase1設計（.cursor/IMPLEMENTATION_PHASES.md など該当箇所）にポリシー追記: 許容リスト3種、リサイズ禁止、設定メニューのみ変更可、起動時はモニタ解像度に収まる最大候補を選択。

2) 起動時の解像度決定

- モニタ解像度を取得し、許容リストから最大で収まる解像度を選択。見つからない場合は最小候補（1280x720）を使用。
- `InitWindow` 前に `SetConfigFlags(FLAG_WINDOW_UNDECORATED?)` は不要。`FLAG_WINDOW_RESIZABLE` を付けない（リサイズ禁止）。
- `SetWindowMinSize` は不要だが、念のため許容リストの最小を設定してもよい。

3) 起動後の振る舞い

- 解像度変更は実装しない（メニュー実装までは固定）。メニュー導入時は許容リストから選択して `SetWindowSize` → `UpdateScreenSize` を呼ぶ方針をコメントで明記。

4) レンダリング/入力

- 仮想解像度は従来どおり 1920x1080。RenderScaled と ScreenToVirtual は現行のレターボックス処理をそのまま使用。

## 完了条件

- 設計ドキュメントにポリシーが明記されている。
- 実行時にウィンドウが許容リスト内の解像度で固定起動し、リサイズ操作を受け付けない。
- レンダリングと入力スケールが従来どおり安定し、UI崩れが再発しない。
