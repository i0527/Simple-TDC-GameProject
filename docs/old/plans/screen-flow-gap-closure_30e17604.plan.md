# 追加明文化プラン（図鑑/ステパネル/入力マップ/キャッシュ）

1) 図鑑の多件数時挙動

- 反映先: `.cursor/ui-system-design.md` 図鑑節。
- 方針: 検索なし。タグ別にグループ化し、各タグ内は名前昇順。件数が多い場合はページング（固定件数、例: 20件/ページ）を採用する、と一文で決める。
- 立ち絵キャッシュ: 遅延ロード＋同時保持は2〜3枚を上限と明記。

2) ステージ詳細/編成パネルのエラー表示と再試行

- 反映先: `.cursor/ui-system-design.md`（ステージ選択/編成節）。
- 方針: 検証エラーは同パネル内トーストで表示、再試行は同パネルから再送可能、と一文で記載。

3) 入力マップ共有の明記

- 反映先: `.cursor/ui-system-design.md` 入力節。
- 方針: ステージ詳細/編成確認はステージ選択の入力マップを共有する、と明記。

4) 立ち絵キャッシュ上限（図鑑）

- 反映先: `.cursor/ui-system-design.md` 図鑑節（キャッシュ/リソース扱い）。
- 方針: 遅延ロード＋同時保持2〜3枚を上限、と明記（1)と併記）。
