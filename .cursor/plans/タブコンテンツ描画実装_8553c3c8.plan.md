# タブコンテンツ描画実装計画

## 概要

現在の`RenderTabContent()`のswitch文を、各タブごとの独立したUpdate/Render関数に分割し、選択されているタブの関数のみを呼び出す形に変更します。

## 実装方針

### 1. 各タブのUpdate/Render関数を追加

各タブ（ホーム、ステージ、編成、強化、図鑑、実績、セーブ）に対して、以下の関数を追加：

- `Update[TabName]Tab(float dt)` - タブ固有の更新処理
- `Render[TabName]Tab()` - タブ固有の描画処理

### 2. `_update()`での呼び出し

`_update()`内で、選択中のタブに応じて対応するUpdate関数を呼び出します。

### 3. `RenderTabContent()`での呼び出し

`RenderTabContent()`内のswitch文を、選択中のタブに応じて対応するRender関数を呼び出す形に変更します。

## 実装詳細

### 追加するメソッド（HomeScene.hpp）

```cpp
// ========== タブコンテンツ更新メソッド ==========
void UpdateHomeTab(float dt);
void UpdateStageTab(float dt);
void UpdateFormationTab(float dt);
void UpdateEnhanceTab(float dt);
void UpdateGalleryTab(float dt);
void UpdateAchievementTab(float dt);
void UpdateSaveTab(float dt);

// ========== タブコンテンツ描画メソッド ==========
void RenderHomeTab();
void RenderStageTab();
void RenderFormationTab();
void RenderEnhanceTab();
void RenderGalleryTab();
void RenderAchievementTab();
void RenderSaveTab();
```

### `_update()`の修正

選択中のタブに応じて対応するUpdate関数を呼び出します。

### `RenderTabContent()`の修正

switch文で選択中のタブに応じて対応するRender関数を呼び出します。

## 利点

- 各タブのロジックが独立して管理できる
- コードの可読性と保守性が向上
- 将来的な機能追加が容易
- タブごとの状態管理が明確になる
