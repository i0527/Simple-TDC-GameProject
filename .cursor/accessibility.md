# アクセシビリティ基本方針

## 1. 色覚コントラスト
- 情報を色のみに依存させない（アイコン/テキストを併用）。
- 主要UIはWCAG AA相当のコントラスト比を目標とする。
- カラーブラインド配慮の配色を優先し、必要に応じて色覚モードを用意する。

## 2. 操作性
- 主要操作にキーボードフォーカスとタブ移動を提供する（エディタ含む）。
- クリック系UIにはフォーカスインジケータを表示する。
- トグル/チェックは状態が明確に分かる文言と見た目を用いる。

## 3. テキスト/フォント
- 読みやすいフォントとサイズ（デフォルト14?16pt相当）を基準とし、スケール可変を想定する。
- テキストはハードコードせず定数/リソース化して差し替え可能にする。

## 4. アニメーション
- 強い点滅や高速なループを避け、フェード/スライド等の穏やかな効果をデフォルトとする。
- アニメーションを無効化/軽減する設定フラグを用意できるよう設計する。

## 5. 音量/フィードバック
- 効果音/音量は調整可能にし、過度な音量差を避ける。
- 成功/失敗のフィードバックは音視覚の両方で提供する。
